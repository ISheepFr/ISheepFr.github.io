Color game

Project written in C++ with OpenCV



Description

This project is a color-based game where the player controls a circle, moving it left or right based on two detected colors. The objective is to eat falling circles to score points before they reach the player’s circle. It involves:

  • Color calibration: Using camera input, the user selects two colors to control the circle’s movement, using a process involving buttons to set pixel thresholds for color detection.

  • Gameplay: The main circle moves based on the presence of each color, with more of one color pushing it left and the other color pushing it right.

  • Ending Condition: The game ends when a falling circle reaches the player's circle without being eaten.


Color calibration

The camera feed is displayed when the game starts. Two buttons, one red and one green, appear at the bottom right for color calibration.

To calibrate the first color, the player clicks on an object in the scene, like a red pen, to select the color. Then, they press the green button to capture the maximum number of pixels of that color. After removing the object, they press the red button to capture the minimum number of pixels.

The same process is repeated for a second color. Once both colors are calibrated, the player can start the game, and the main circle moves based on the detection of those two colors in the scene.
