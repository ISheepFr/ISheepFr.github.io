Polyhedra fusion

Project written in C++ 14

View on GitHub

Description

This project aims to develop an algorithm for iteratively merging elementary convex polyhedra in order to minimize the number of resulting convex polyhedra.

The objective is to determine the optimal order in wich to merge the polyhedra to minimize this number.

The merging program reads a 3D file in *.obj format as input, merges the polyhedra together as much as possible, and writes a corresponding *.obj file that can be viewed, for example, in Blender.

The implemented features are listed below :

  • Polyhedra fusion
  • Convex fusion graph
  • Brute force, simulated annealing & genetic algorithm
  • Saving/Displaying Results


Read 3D Object

The reading function allows you to read a 3D file in *.obj format, dynamically create objects in C++, and store the objects in the file using the obj syntax.

Each line of the file is processed, and an action is performed based on its associated keyword.

The main syntax for a 3D object is as follows: it includes the vertices, the object name, and the object's faces, along with the vertex numbers that form each face.

Dynamic creation and convexity

Once the objects are created, the convexity of each polyhedron is tested and marked. After reading, the list of polyhedra, faces, and vertices is retrieved and stored in the algorithm class.

We now apply one of the algorithms we developed to perform the polyhedron permutations. The permutation that yields a valid solution with the fewest output polyhedra is chosen. The goal is to always lower the final number of polyhedra after the merge.

Write 3D Object

Finally, the solution(s) are written in *.obj format, passing the filename, the list of vertices, and the list of polyhedra as parameters. First, all the vertices of the solution are written, then all the polyhedra. The solution(s) can then be visualized, for example, in Blender.
