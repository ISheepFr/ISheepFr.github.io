Protoype of a Vehicule 2 Vehicule connexion simulation

Project written in C++, QML, with QT and OpenStreetMap



Description

This project visualizes vehicle-to-vehicle (V2V) communication in the context of connected vehicles using 5G technology. The tool simulates vehicle movement and communication propagation over an OpenStreetMap-based environment.

  • Hexagonal Grid: The environment is structured using a grid of hexagonal cells for accurate communication modeling.

  • Vehicle Simulation (Prototype): Users can configure vehicle properties such as speed, emission power, and communication frequency, but many functions are not yet implemented, making this a prototype.

  • Dynamic Visualization: Real-time updates of vehicle movement and communication status on the map.

C++ Code Organization

The C++ part of this project handles the core simulation logic, focusing on the creation of the environment, vehicles, and communication modeling. The code is organized into various classes to ensure modularity and clarity.

  • Hexagon Class: Defines each hexagonal cell's attributes, such as position, neighbors, and color for the simulation grid.

  • Grid Class: Manages the entire grid of hexagons, including functions for creating the grid and accessing neighboring hexagons for communication purposes.

  • Vehicle Class: Represents individual vehicles with properties like position, speed, emission power, and movement options. This class serves as the foundation for vehicle interaction and communication.

  • Main C++ File: Converts the C++ classes into QML-compatible objects, enabling their use in the graphical interface.

QML Integration and Usage

The QML layer of this project provides a user-friendly interface for interacting with the vehicle-to-vehicle (V2V) communication simulation. It integrates with the C++ backend to utilize the simulation logic.

  • Main QML File: Acts as the central point for the application layout, displaying the simulation grid and components, while facilitating user input for configuration.

  • HexagonItem.qml: Responsible for rendering each hexagonal cell on the grid, utilizing properties defined in the C++ Hexagon class for visual representation.

  • VehicleItem.qml: Displays vehicles on the grid, leveraging data from the C++ Vehicle class to dynamically show their positions and states in the simulation.

  • Function Calls: QML can directly invoke C++ functions to modify vehicle parameters and retrieve simulation data, ensuring a smooth interaction between the visual interface and the underlying logic.
